## Work Packages

### 1. Sensing and Input Working Package

**Weeks 1-3:** Camera handling, preprocessing, noise cancelling, ROIs definition

**Weeks 4-6:** Define use-case and test given servers information (localisation on map, cars interaction, gps interaction)

**Weeks 7-11:** Define other necessary sensors, define use-case, integration (IMU, distance), preprocessing, noise cancelling

**Weeks 12-14:** Induce noise on all sensors and systems

**Weeks 15-25:** Other functionalities and optimizations

---

### 2. Perception and Scene Understanding Working Package

**Weeks 1-2:** 
- Documentation on the given guides and projects
- Chose main languages and technologies
- Create/adapt project plan
- Members tasks assignation

**Weeks 3-4:** Lane detection

**Weeks 5-6:** Intersection detection

**Weeks 7-8:** Traffic sign detection

**Weeks 9-10:** Traffic light detection

**Week 11:** Position fusion

**Weeks 12-13:** 
- Define objects properties file
- Environmental server interaction

**Weeks 14-15:** 
- Traffic lights detection & classification
- Object detection & classification

**Weeks 16-25:** Other functionalities and optimizations

---

### 3. Behaviour and Motion Plan Working Package

**Weeks 3-6:** Define project architecture and communication between packages

**Weeks 7-9:** Define path planning and validation

**Weeks 10-12:** 
- Define robustness and safety measures
- Define decision making --> priorities of actions and state flow

**Weeks 13-15:** Induce noise on systems to validate robustness (loss of image, burned image, road search, undefined objects and states)

**Weeks 16-25:** Other functionalities and optimizations **BFM+**

---

### 4. Vehicle Control Working Packages

**Weeks 3-7:** Lane following and speed control

**Weeks 8-12:** Intersection navigation

**Weeks 13-15:** Simple action taking maneuvers (parking, stop for traffic sign, stop for traffic light, stop for pedestrian)

**Weeks 16-18:** Complex action taking maneuvers (switch lane for static and mobile car, road search)

**Weeks 19-25:** Other functionalities and optimizations

---

### 5. Final Result & Demo

**Weeks 1-2:** Team can control the car remotely and the visual car on the simulator

**Weeks 3-6:** 
- Robot can keep a lane, can make a curve
- Team defines and creates its own physical testing environment
- Team installs the virtual testing environment

**Weeks 7-9:** Team defines a way of parallel developing and testing

**Weeks 10-14:** Robot can navigate in intersection

**Weeks 15-18:** Robot can go on a pre-determined path, stop at stop sign, park at parking sign, slow at crosswalk

**Weeks 19-22:** While detecting and calculating its position, the robot can dynamically go to specified checkpoint, react to traffic lights, interact with other cars and send environment data

**Weeks 23-25:** Other functionalities and optimizations
